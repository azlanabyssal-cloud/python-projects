Exploratory Data Analysis

Before you train a single model, you must understand your data. EDA is where bad assumptions get caught — and good features get discovered.

Layer 1 — Simple & Easy understand the core habit in 60 seconds
The First Rule of ML

Look at your data before you model it. Every time.

EDA is the process of understanding your dataset before feeding it to any algorithm. You look at its shape, its distributions, its missing values, and its relationships. Without EDA, you are training blind.

A senior ML engineer at Flipkart or Google spends 1–2 days on EDA before writing a single line of model code. EDA catches data leakage, outliers, wrong dtypes, skewed distributions, and multicollinearity — all of which silently destroy model quality if missed.

Layer 2 — Master in Real the four things EDA actually reveals
Distribution Shapes

The shape of your data tells you which statistics to use.

Before computing mean and std, ask: what shape is this distribution? A right-skewed salary distribution makes the mean meaningless — and your linear model's assumptions invalid. Toggle the shapes to see the four patterns you will encounter in every real dataset.

Key EDA Metrics

Three numbers every ML engineer quotes in interviews.

IQR
Q3 − Q1
Spread of the middle 50%. Outlier fences: Q1 − 1.5×IQR (lower) and Q3 + 1.5×IQR (upper). Robust to extremes — unlike std, which is dragged by the same outliers you are trying to detect.
Skewness
0 = symmetric · + = right tail · − = left tail
|skewness| > 1 → significantly non-normal. Use median for central tendency. Apply np.log1p() transform to pull right-skewed distributions toward normality.
Pearson r
−1 to +1 · linear relationship only
|r| > 0.8 = strong. |r| 0.4–0.8 = moderate. |r| < 0.4 = weak. For non-linear relationships, use Spearman rank correlation instead. High r between two input features = multicollinearity warning.
EDA Code

Three levels of EDA — all using pandas and seaborn.

import pandas as pd df = pd.read_csv('swiggy_orders.csv') # First look — always run these three print(df.shape) # (100, 8) — rows, columns print(df.dtypes) # delivery_time: float64 print(df.isnull().sum()) # missing count per column # Summary statistics — your first sanity check print(df.describe()) # delivery_time rating order_value # count 100.000000 98.0000 100.000000 # mean 34.200000 4.1000 482.000000 # std 11.300000 0.6200 198.000000 # min 14.000000 1.8000 80.000000 # 25% 26.000000 3.7000 320.000000 # 50% 32.000000 4.2000 450.000000 # 75% 41.000000 4.6000 610.000000 # max 9999.000000 5.0000 1240.000000 # ⚡ — investigate this immediately
import matplotlib.pyplot as plt import seaborn as sns fig, axes = plt.subplots(1, 3, figsize=(14, 4)) # Distribution shape — is it normal, skewed, bimodal? sns.histplot(df['delivery_time'], kde=True, bins=20, ax=axes[0]) axes[0].set_title('Delivery Time Distribution') # Boxplot — outliers appear as individual dots sns.boxplot(x=df['delivery_time'], ax=axes[1]) axes[1].set_title('Boxplot — Outlier Detection') # Correlation heatmap — which features move together? corr = df[['delivery_time','rating','order_value']].corr() sns.heatmap(corr, annot=True, fmt='.2f', cmap='coolwarm', center=0, ax=axes[2], square=True) axes[2].set_title('Correlation Matrix') plt.tight_layout() plt.show()
import pandas as pd import numpy as np def eda(df, target=None): print("── Shape ──────────────────────────────") print(f" Rows: {df.shape[0]:,} Columns: {df.shape[1]}") print("\n── Missing Values ─────────────────────") miss = df.isnull().sum() miss = miss[miss > 0] print(" None" if miss.empty else miss.to_string()) print("\n── Summary Stats ──────────────────────") print(df.describe().round(2).to_string()) print("\n── Outliers (IQR method) ──────────────") for col in df.select_dtypes(include=np.number).columns: Q1 = df[col].quantile(0.25) Q3 = df[col].quantile(0.75) fence_hi = Q3 + 1.5 * (Q3 - Q1) fence_lo = Q1 - 1.5 * (Q3 - Q1) n = ((df[col] > fence_hi) | (df[col] < fence_lo)).sum() if n: print(f" {col}: {n} outliers") if target: print(f"\n── Correlation with '{target}' ────────") print(df.corr()[target].sort_values(ascending=False).round(3).to_string()) eda(df, target='rating')

The Interview Questions

5 questions. Exact scripts. What they are actually testing.

These are EDA questions asked at the companies above. Each answer is a 30–45 second script. Click to expand.

Q1 Walk me through how you would do EDA on a new dataset you have never seen before. ▶
All Tiers Tests: do you have a repeatable process?
What they want: A systematic process, not ad-hoc exploration. They want to see you have a mental checklist.
Say exactly this (40 seconds)
"I start with shape and dtypes — how many rows, columns, and what type is each feature. Then I run isnull().sum() to map the missing data. Then describe() to see count, mean, std, and especially the min and max — the max is where outliers hide. I look for anything that breaks common sense: negative ages, delivery times of 9999 minutes, salary of 0. Then I plot histograms for each numeric feature — this reveals skewness and bimodal distributions that describe() cannot. Finally I build a correlation matrix to spot multicollinearity and find which features actually correlate with the target. The whole process takes 1–2 hours. It saves days of debugging later."
Follow-up they will ask: "What automated tools do you know?" — Answer: ydata-profiling (formerly pandas-profiling) generates a complete HTML report in one line: ProfileReport(df).to_file("report.html"). Also sweetviz for comparing train vs test distributions. These are for quick audits — they do not replace manual EDA for domain-specific insight.
Q2 The mean delivery time is 45 min but the median is 28 min. What does this tell you? ▶
India Product · MNC Product Tests: do you understand skewness?
What they want: You to name the shape (right-skewed), explain why it happens, and know the correct measure of central tendency.
Say exactly this (35 seconds)
"Mean of 45, median of 28 tells me the data is right-skewed — a handful of extreme late deliveries are pulling the mean up, but most deliveries actually complete around 28 minutes. The median is the honest number here. I would report 'typical delivery time: 28 min' to the business, not 45. I would also investigate what's causing those extreme values — data errors, special order types, specific locations? And before feeding this to a linear model, I would apply a log transform to pull the distribution toward normality."
Follow-up they will ask: "How do you quantify skewness?" — Answer: df['delivery_time'].skew(). Positive value = right-skewed. Rule of thumb: |skewness| > 1 is significantly non-normal. Between 0.5 and 1 is moderate. Below 0.5 is roughly symmetric.
Q3 How do you detect outliers? And what do you do after detecting them? ▶
India Product · Big Tech Tests: detection AND decision process
What they want: Two-phase answer — detect first, decide second. Most candidates only answer half.
Say exactly this (45 seconds)
"I use the IQR method: compute Q1 and Q3, then IQR = Q3 − Q1. The fences are Q1 − 1.5×IQR (lower) and Q3 + 1.5×IQR (upper). Anything outside these fences is a candidate outlier. I also use boxplots visually — the dots outside the whiskers are the outliers. But detecting is step one. Step two is understanding WHY: is it a data entry error like a delivery time of 9,999 minutes? A genuine extreme case — a 3-hour delivery to a remote location? Or a system placeholder — 0 or 9999 meaning 'unknown'? The answer determines the fix: cap it at the fence, set it to NaN and impute, add a binary 'was_outlier' flag, or keep it if it's real. I never drop outliers without understanding what they represent."
Follow-up they will ask: "When would you use z-score instead of IQR?" — Answer: z-score (|z| > 3) works well for normally distributed features. IQR is better for skewed distributions — it is not pulled by the same extreme values you are trying to flag. In practice, use IQR by default unless you have confirmed normality.
Q4 Correlation between study_hours and first_salary is 0.84. Does more studying cause higher salary? ▶
India Product · Big Tech Tests: do you confuse correlation with causation?
What they want: You to firmly say no — and explain confounders. This is a classic trap.
Say exactly this (35 seconds)
"No. Correlation of 0.84 means they move together, but it does not mean studying causes the salary. A confounder could explain both — students who study more may also attend better colleges, live in metro cities, or have higher conscientiousness — all of which independently raise salary. The ice cream and drowning deaths example: both correlate, both are driven by summer heat, neither causes the other. To establish causation I would need a controlled experiment — randomly assign study hours to comparable students. In ML, I use this correlation as a signal that study_hours is a useful feature — I report it as 'associated with', not 'causes'."
Follow-up they will ask: "What's the difference between Pearson and Spearman correlation?" — Answer: Pearson measures the strength of a LINEAR relationship. Spearman measures whether as one variable goes up, the other tends to go up — regardless of whether that relationship is linear. Use Spearman when features have outliers or non-linear relationships.
Q5 Two features in your dataset are correlated at r=0.97. What do you do? ▶
India Product · Big Tech Tests: do you know multicollinearity and when it matters?
What they want: You to know that multicollinearity matters for linear models but not trees — and offer concrete fixes.
Say exactly this (40 seconds)
"r=0.97 is very high multicollinearity — the two features carry nearly identical information. My response depends on the model. For a linear or logistic regression, this inflates the variance of both coefficients — small changes in data cause large swings in the coefficients, making the model unstable and uninterpretable. I check VIF: if VIF is above 10, I drop one feature or use Ridge regression which handles this via L2 regularisation. For tree models — Random Forest, XGBoost — multicollinearity does not affect predictive performance but it can mislead feature importance scores. Both features split credit between them, so I might compute SHAP values instead of raw importances to get the true contribution."
Follow-up they will ask: "What is VIF?" — Answer: Variance Inflation Factor. VIF of 1 = no multicollinearity. VIF of 5 = moderate. VIF > 10 = high — take action. Computed as variance_inflation_factor(X.values, i) from statsmodels for each feature i.
